🔄 Full Circle: From AI-Incident Investigation to an Auto-Generated Sentinel Playbook — All in Natural Language

In a single afternoon, I went from a raw AI-attack alert in Microsoft Sentinel to a production-ready response playbook. Without writing a single line of KQL or Python by hand.

Here's how ↓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔍 THE INCIDENT

An Entra user started receiving a burst of AI-targeted alerts over 6 days:

→ ASCII smuggling prompt injection (×2)
→ AI jailbreak attempts blocked by Prompt Shields (×2)
→ User phishing on AI agents (×2)
→ AI Agent Reconnaissance
→ 3× Purview IRM "Risky AI usage"
→ DLP "Restrict copilot by label"

Three attack vectors. One user. Same Azure OpenAI deployment. This was a campaign, not a single alert.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🧠 STEP 1 — INVESTIGATE FROM VS CODE

I opened VS Code with CyberProbe (my open-source AI investigation platform) and typed one line to GitHub Copilot:

"Auto-investigate the user with the most AI-attack alerts in the last 7 days."

Behind the scenes, MCP servers chained automatically:
→ Sentinel Data Lake — 30d baseline (473 sign-ins, 1 IP)
→ Defender XDR Triage — incident details, MITRE
→ Microsoft Graph — user profile, OAuth resolution
→ Microsoft Learn — current Prompt Shields docs

Key finding: the attacker IP did NOT appear in the user's 30-day Entra sign-in history. The AI API was being invoked OUTSIDE the Entra auth flow = stolen API key or exfiltrated session token. Not a compromised identity.

Cluster correlation: 6 users sharing the same OAuth object ID on the same Azure OpenAI deployment. Systemic abuse, not isolated.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 STEP 2 — THE REPORT

CyberProbe produced an HTML report with a dark-theme Mermaid attack-path graph, MITRE mapping, enriched IP tables, and a mandatory Methodology section citing every query and tool.

Full report (sanitized, public): github.com/olchar/CyberProbe/tree/main/reports

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🤖 STEP 3 — SECURITY COPILOT REASONING

With Security Copilot (E5/E7 AI capacity benefits), I promoted the incident into a guided investigation. Embedded AI produced an executive summary, reframed MITRE tactics into plain-English business risk, and suggested a prioritized response plan.

This is where "AI for Security" earns its keep: compressing time from findings to executive decision.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚙️ STEP 4 — CODIFY AS A PLAYBOOK

Microsoft recently shipped the Sentinel Playbook Generator (preview) — a Cline-based AI coding agent embedded in the Defender portal. You write natural language, it writes Python.

So I wrote a paste-ready spec reproducing the investigation:
→ Trigger on AI-targeted alert
→ Profile user baseline via runHuntingQuery
→ Check ai_api_bypass (attacker IP NOT in baseline)
→ Parallel IP enrichment (AbuseIPDB + IPInfo + VPNapi)
→ OAuth GUID resolution
→ Cluster correlation across users
→ 3-tier risk scoring
→ Post Markdown comment
→ GATED response — never auto-disable or isolate

I pasted the spec into Plan mode → reviewed the flow diagram → switched to Act mode. The generator wrote the Python, the integration bindings, and the test harness.

⚠️ Important: the generator writes code. It does NOT tell you whether the code is safe.

So the spec includes a 14-item validation checklist across 5 categories — Security / Reliability / Blast-radius / Observability / Regression. Treat every generated playbook as untrusted until it passes. Same discipline as reviewing a PR from a new team member — except the team member is an LLM.

One investigation → reusable infrastructure.
