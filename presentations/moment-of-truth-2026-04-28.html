01 · The question

The reality nobody puts on a vendor slide

Multiple clouds

Azure + AWS + GCP. Each with its own console, its own scoring, its own alerts.

Multiple stacks

EDR · CSPM · CNAPP · Identity · Email · DLP — different vendors, different taxonomies.

Multiple scanners

Qualys · Tenable · MDVM · container · IaC · DAST · SAST — each with its own severity model.

02 · The reality

This is a moment of truth.

Consoles, SaaS, and dashboards are still relevant.
For how long?

The adversary isn't opening 15 tabs. Neither should we.

Same signal. Four audiences. Four unmet needs.

CISO / VP Security

Asks: Are we improving? What's residual risk? Can I defend it to the board?

Gets: 47 dashboards, irreconcilable scores.

Security Architect

Asks: Where are the exploitable paths? Which choke points matter most?

Gets: 3,000 CVEs ranked by CVSS, no graph context.

Project Manager

Asks: What do we prioritize? What's the effort and timeline?

Gets: Unusable lists. Spreadsheets. Monthly.

Infra / Ops

Asks: What do I patch, on which host, by when?

Gets: CVEs without business context.

03 · The consolidation gap

The shift: from consoles to an investigation fabric

Yesterday — Console-centric

  • Humans pivot between 15 UIs
  • Context lost at every tab
  • Every vendor wants to be the "single pane"
  • Data pulled into consoles
  • Value = the screen

Tomorrow — Agent-centric

  • One analyst, one prompt, many MCP tools
  • Context carried through every step
  • The data plane is the platform
  • Agents pulled to the data
  • Value = the outcome

Consoles aren't dying tomorrow. But they're being demoted — from the workplace to a view over the fabric.

Demo · Part 1 — Investigate from VS Code

Prompt:

Auto-investigate the user with the most AI-attack alerts in the last 7 days.
  1. Classify entity — UPN, last-30d baseline via SigninLogs (473 sign-ins, 1 IP).
  2. Pull incidents & alerts — Defender XDR Triage MCP: 8 AI-targeted alerts, 3 attack vectors.
  3. Enrich IPs in parallel — AbuseIPDB · IPInfo · VPNapi · Shodan.
  4. Phase 2 finding — attacker IP not in 30-day Entra sign-in history → ai_api_bypass = True.
  5. Phase 3 cluster — 6 users sharing one OAuth object ID on the same Foundry deployment.

That one fact collapsed the whole case: stolen API key, not compromised identity.

08 · Demo · codify

What this means for the SOC

Be faster

Hour-long investigations collapse to minutes. Triage parallelizes. Baselines are automatic.

Be assisted

Billions of log rows · millions of sign-ins · thousands of alerts — beyond human scale. AI isn't optional; it's the only way to cover the ground.

Be scalable

One analyst + agents covers what a five-person team used to. Expertise gets encoded, not just remembered.

Be layered

H2A and A2A. Humans set intent; agents execute. One-way doors stay gated by humans — always.

The adversary is already operating this way. The question is how fast defenders close the gap.

09 · SOC operating model

What this means for vendors — and buyers

If your product is a console…

  • …it competes with 14 others for attention.
  • Value decays as agents bypass UIs.
  • The "single pane of glass" promise expired.

If your product is a data & control plane…

  • Expose clean APIs, MCP servers, graph context.
  • Let agents orchestrate through you.
  • Stickiness = schema + signal quality, not UI.
Buyers: stop asking "show me the dashboard."
Start asking "show me the API, the MCP tool, and the schema."
