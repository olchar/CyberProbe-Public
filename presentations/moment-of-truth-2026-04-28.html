The shift: from unified console to investigation fabric

Today — Unified XDR console

Defender XDR already solved the “15 tabs” problem.

  • Identity + endpoint + email + cloud apps correlated in one portal
  • Copilot-in-portal assists on the current incident
  • Reasoning scoped to what the vendor ingests
  • Third-party signals need a connector — or they don't show up
  • Expertise lives in the analyst's head and dies when the tab closes

Tomorrow — Agentic fabric

The conversation lives outside any one console.

  • One prompt orchestrates XDR, Sentinel, Graph, and third-party MCPs as peers
  • Context, queries, and IOCs travel across products
  • Campaign-correlation across 90 days and multiple workspaces
  • Every investigation compounds into code — reports, playbooks, detections
  • Humans stay on the one-way doors; agents do the heavy lifting

The real differentiator isn't single pane of glass. It's whether expertise compounds into code or evaporates into a UI session.

01 · The shift
Open with a correction of a lazy framing: the “15 consoles” critique is a 2020 story. Defender XDR already consolidated identity, endpoint, email, cloud apps and cloud into one portal, with correlated incidents, a unified entity page, one hunting surface, and Copilot-in-portal assistance. Credit that honestly — the room is full of people who shipped it. Then land the real 2026 gap. Four points: (1) the unified console is still a GUI — expertise lives in analysts’ heads, not in reusable code; (2) the workload is bounded by what the vendor ingests — third parties need connectors or stay invisible; (3) a great investigation ends when the tab closes — nothing becomes a playbook, nothing becomes a regression test; (4) reasoning is per-incident, not campaign-wide across 90 days and multiple workspaces. Punch line: the real differentiator is not single pane of glass. It’s whether expertise compounds into code or evaporates into a UI session. Lead into slide 4, where the three postures (SaaS-only, SaaS+Copilot, Agentic) make this operational.

Demo · Part 1 — Investigate from VS Code

Prompt:

Auto-investigate the user with the most AI-attack alerts in the last 7 days.
  1. Classify entity — UPN, last-30d baseline via SigninLogs (473 sign-ins, 1 IP).
  2. Pull incidents & alerts — Defender XDR Triage MCP: 8 AI-targeted alerts, 3 attack vectors.
  3. Enrich IPs in parallel — AbuseIPDB · IPInfo · VPNapi · Shodan.
  4. Phase 2 finding — attacker IP not in 30-day Entra sign-in history → ai_api_bypass = True.
  5. Phase 3 cluster — 6 users sharing one OAuth object ID on the same Foundry deployment.

That one fact collapsed the whole case: stolen API key, not compromised identity.

Beyond investigation — where agentic goes next

Investigating faster is the no-brainer. The real leverage is everywhere else we still move slowly.

🔎 Investigate faster

The baseline win. Hours → minutes. Triage, enrichment, correlation in parallel.

🩹 Patch faster

Gather vuln context → map to exposure graph → prioritize by blast radius → open change tickets → track remediation. Today it's weeks. It shouldn't be.

🧱 Harden faster

Detection-as-code. Sigma → KQL → analytics rule → deployed. Community intel lands in your tenant the same day.

📋 Report faster

ITDM briefings, board decks, compliance evidence — persona-specific outputs from the same fabric.

🏗️ Build faster

New MCP servers, new skills, new connectors — agents help you extend the fabric itself.

🤝 Collaborate faster

H2A + A2A. Hand an investigation to another agent, to a peer, to Security Copilot — with full context.

10 · Beyond investigation
Open the aperture. Investigating faster is the obvious win and the one every vendor demos. The real leverage is in everything we still move slowly on: patching, hardening, reporting, building, collaborating. Emphasize patching and hardening — that’s where agentic reaches Change Management, Vuln Management, Detection Engineering. Reporting persona-specific outputs is where the ITDM vs SOC conversation from the Hydro-Québec slide becomes operational. Say clearly: agentic has to be part of every step, not just triage.

On the defensive side — we need

If the offense is agentic end-to-end, a defense that's agentic only in Phase 1 is already losing.
11 · The asymmetry
Bring urgency without fear-mongering. The offensive side has purpose-built models — Claude Mythos-class, GPT-5.4-Cyber — automating recon, exploit chaining, and phishing at machine speed. If we match that only in Phase 1 triage and leave the rest manual, the asymmetry wins. The defensive answer isn’t more tools. It’s agentic coverage end-to-end: gather faster, prioritize faster, patch and contain faster. Pull quote to land: “if the offense is agentic end-to-end, a defense that’s agentic only in Phase 1 is already losing.”

In a world of humans and agents — what comes next?

CyberProbe shows how agents investigate. The next question is how you operate them when there are thousands of them working next to your people.

Agents need the same fabric humans already have

  • Identity — a real identity per agent, not a shared key
  • Access — least-privilege, conditional, auditable
  • Security & governance — the same controls as any workforce
  • Lifecycle — onboarding, review, offboarding
  • Compliance — discoverable, auditable, explainable

Microsoft's answer: Agent 365

  • Agents as first-class identities in Entra
  • Governed, monitored, and managed like any worker
  • Built into the same platform your humans already run on
