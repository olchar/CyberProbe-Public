🛡️ CNAPP Coverage Model — CSPM + CWPP

CNAPP = CSPM + CWPP. All 13 asset types receive Defender CSPM coverage (posture assessment, misconfiguration detection, attack paths, security graph). Assets with supported runtime agents also receive CWPP workload protection (threat detection, EDR, vulnerability scanning).

📐 Calculation Rationale — How Each Value Was Derived

1 — Defender for Servers P2: $19,282/mo (CAD)
Input: 964 EC2 instances (customer-provided)
Calculator field: "Servers outside Azure" → enter 964 servers
Calculation: 964 servers × $20.00/server/month = $19,282
Unit price derivation: Azure Retail Prices API (CAD, Canada Central): $0.0274/server/hour. At 730 hours/month: $0.0274 × 730 = $20.00/server/month
EBS disks (7,248): Not separately billable. Agentless disk scanning and malware detection are bundled with the Servers P2 license covering the parent VM. No additional line item.
Assumption: All 964 VMs are in running state. Deallocated/stopped VMs are excluded from billing per Microsoft Learn.
2 — Defender for Containers: TBD
Input: 1 EKS cluster + 1,797 ECR container images (customer-provided)
Calculator field: "Container — vCores" → enter the total vCore count across all EKS worker nodes
Calculation: [vCores TBD] × $9.42/vCore/month = TBD
Why vCores, not nodes: Defender for Containers bills by vCore, not by node or cluster. The customer must provide the total vCore count across all worker nodes in the EKS cluster.
Unit price derivation: Azure Retail Prices API (CAD, Canada Central): $0.0129/vCore/hour (vCore Pack). At 730 hours/month: $0.0129 × 730 = $9.42/vCore/month
Image scanning: 20 free vulnerability assessments per charged vCore per month. If the cluster has ≥90 vCores → all 1,797 images are covered free (90 × 20 = 1,800). Otherwise, overage is $0.40/image digest.
ECS Fargate (1,320 tasks): Serverless — not covered by Defender for Containers runtime protection (DaemonSet requires worker nodes). Covered by CSPM posture only. Not billable under Containers plan.
3 — Defender for Databases: $7,183/mo (CAD)
Input: 200 RDS instances + 150 Aurora instances = 350 total (customer-provided)
Calculator fields: In the Azure Calculator under "Defender for open-source relational databases", use:
• MySQL: Enter count of MySQL-engine RDS + Aurora MySQL-compatible instances
• PostgreSQL: Enter count of PostgreSQL-engine RDS + Aurora PostgreSQL-compatible instances
• MariaDB: Enter count of MariaDB-engine RDS instances (if any)
Why MySQL/PostgreSQL: Both are priced at $20.52/instance/month (CAD) in the Azure Calculator. The total cost is identical regardless of how you split them (350 × $20.52 = $7,183 either way). We used the combined count because the customer did not provide the per-engine breakdown.
⚠️ SQL Server engine exception: If any RDS instances run SQL Server engine, those should go under "SQL Database outside Azure" at $0.0276/vCore/hour (~$20.15/vCore/month CAD). This is a vCore-based model, not per-instance — requires knowing the vCore count of those specific instances.
"SQL Database on Azure" ($20.52/server/mo CAD): This field is for Azure SQL Database only — not applicable to AWS RDS.
DynamoDB (369 tables): NoSQL database — not covered by Defender for Databases (which targets relational databases). Also not a billable CSPM resource. Benefits from CSPM posture features at no additional charge.
Aurora billing note: Aurora instances are billed per instance, same as RDS. Aurora is MySQL/PostgreSQL-compatible, so it maps to those calculator fields.
4 — Defender CSPM: $38,581/mo (CAD)
Input: 964 EC2 + 4,204 S3 + 350 RDS = 5,518 billable resources
Calculator field: "Defender CSPM" → enter 5,518 billable resources
Calculation: 5,518 × $7.01/resource/month ≈ $38,581 (Azure Calculator shows $38,580.52)
Unit price derivation: Azure Retail Prices API (CAD, Global): $0.0096/resource/hour. At 730 hours/month: $0.0096 × 730 = $7.008 ≈ $7.01/resource/month
AWS billable resource types (per Microsoft Learn):
• Compute: EC2 instances (excl. deallocated) — 964
• Storage: S3 buckets — 4,204
• Databases: RDS instances (incl. Aurora) — 350
NOT billable CSPM resources: Lambda (4,910), ECS Fargate (1,320), DynamoDB (369), EBS disks (7,248), asset metadata (3,421), developers (406), CNAPP users (30). These benefit from foundational and/or paid CSPM features but are not counted in the billable resource total.
CSPM is billed independently from CWPP plans: EC2 instances are billed under both Servers P2 ($20.00 CAD) and CSPM ($7.01 CAD). These are separate, additive charges — enabling Servers P2 does not include or replace the CSPM charge. Foundational CSPM (free) provides basic recommendations and secure score; Defender CSPM (paid) adds attack paths, security explorer, regulatory compliance, governance, and data-aware posture.
⚠️ Upcoming change (April 1, 2026): AWS Lambda functions will become billable CSPM resources. Estimated impact: ~4,910 functions ÷ 8 (resource grouping ratio per MS Learn) ≈ 614 billable resources × $7.01 = +$4,304/mo CAD. Not included in current subtotal.
5 — GitHub Advanced Security: $27,202/mo (CAD est.)
Input: 406 developers (customer-provided)
Calculator: Not in Defender for Cloud calculator. Use Azure DevOps pricing page.
Calculation: 406 active committers × ~$67/committer/month (CAD est.) = $27,202
Unit price note: The USD list price is ~$49/committer/month. The CAD estimate (~$67) is derived from the same exchange ratio observed in the Defender for Cloud calculator (×1.37). Confirm the exact CAD rate on the Azure DevOps pricing page with CAD currency selected.
Billing model: GHAzDO charges per active committer — a unique user with ≥1 push to a GHAzDO-enabled repo in the billing cycle. The 406 figure assumes all developers are active committers; actual count may be lower.
What's included: CodeQL code scanning, secret scanning, dependency scanning, and push protection.
DevOps posture features: Attack paths, security explorer, and PR annotations for IaC findings require Defender CSPM (Plan #4) in addition to GHAzDO. These are CSPM features, not GHAzDO features — but they need both plans enabled to work together.

CSPM / CWPP Coverage Analysis (March 25, 2026)

  • All 13 asset types receive Defender CSPM coverage — posture assessment, misconfiguration detection, attack path analysis, security graph
  • CWPP runtime protection available for: EC2 VMs (Servers P2), EKS (Containers), ECR images (Containers), RDS/Aurora (Databases), EBS (bundled), Developers (GH Advanced Security)
  • CSPM-only assets (no CWPP runtime agent): S3 buckets, Lambda functions, ECS Fargate — verified against Microsoft Learn support matrices
  • Defender for Storage: docs confirm Azure Blob/Files/Data Lake only — S3 covered by CSPM posture
  • Defender for Servers: overview confirms VMs on Azure, AWS EC2, GCP Compute — Lambda covered by CSPM posture
  • Defender for Containers: support matrix lists AKS, EKS, GKE, Arc-enabled K8s — Fargate covered by CSPM posture

Plan Assignment Logic

  • Each asset category mapped to its CWPP plan where runtime agent deployment is supported, plus Defender CSPM for all assets
  • Assets where CWPP agents cannot be deployed (serverless, non-Azure storage) are covered by CSPM posture management
  • CSPM billable resources limited to EC2 instances, S3 buckets, and RDS instances per Microsoft Learn documentation
  • DynamoDB classified as non-billable CSPM resource — NoSQL, no SQL threat detection sub-plan applies
  • Non-OS disks (EBS) mapped to Defender for Servers — disk exposure context is bundled, not separately billed
  • Developer seats mapped to both Defender CSPM (DevOps posture) and GitHub Advanced Security (code scanning)
  • CNAPP users (30) flagged as non-billable — portal access, not a metered resource
